ІНСТРУКЦІЯ ДО TELEGRAM-БОТА СИСТЕМИ «ADHD» (Alert Drago Help Desk)

Цей документ описує всі команди, кнопки та робочі процеси Telegram-бота
для технічного персоналу кафедри.

═══════════════════════════════════════════════════════════════════
1. ЗАПУСК ТА ПОЧАТОК РОБОТИ
═══════════════════════════════════════════════════════════════════

1.1. Знайдіть бота в Telegram та натисніть /start.

1.2. Бот відповість привітанням та покаже головне меню
     з inline-кнопками для перегляду заявок.

1.3. Доступ мають лише авторизовані користувачі, чиї Telegram ID
     внесені до змінної ALLOWED_TELEGRAM_IDS у docker-compose.yml.

═══════════════════════════════════════════════════════════════════
2. КОМАНДИ БОТА
═══════════════════════════════════════════════════════════════════

/start
  Показує головне меню з кнопками:
  • 🆕 Нові       — список нових заявок
  • 🔧 В роботі   — заявки, що виконуються
  • ✅ Виконано    — завершені заявки
  • 🔒 Закрито     — відхилені/закриті заявки
  • 📂 Мої заявки  — заявки, призначені на вас

/tickets
  Дублює головне меню /start — показує ті ж inline-кнопки
  для швидкого доступу до списків заявок.

/ticket <номер>
  Запит повної інформації про заявку за її номером.
  Приклад: /ticket 5

  Відповідь містить:
  • Номер заявки
  • Аудиторію
  • Ім'я заявника
  • Опис проблеми
  • Поточний статус
  • Виконавця (або «Не призначено»)
  • Дату створення

═══════════════════════════════════════════════════════════════════
3. INLINE-КНОПКИ ГОЛОВНОГО МЕНЮ
═══════════════════════════════════════════════════════════════════

🆕 Нові
  Показує список усіх заявок зі статусом «Нова».
  Це заявки, які щойно надійшли і ще не взяті в роботу.

🔧 В роботі
  Показує заявки, які наразі виконуються кимось із персоналу.

✅ Виконано
  Показує завершені заявки.

🔒 Закрито
  Показує заявки, відхилені усіма виконавцями.

📂 Мої заявки
  Показує лише ті заявки, які призначені саме на вас.
  Фільтрація відбувається за вашим Telegram ID.

У кожному списку відображаються:
  • Номер заявки
  • Аудиторія
  • Статус
  • Короткий опис (до 80 символів)
  • Загальна кількість заявок

Для перегляду деталей конкретної заявки зі списку
використайте команду /ticket <номер>.

═══════════════════════════════════════════════════════════════════
4. ОБРОБКА НОВИХ ЗАЯВОК (СПОВІЩЕННЯ)
═══════════════════════════════════════════════════════════════════

Коли користувач створює заявку через веб-сайт, бот автоматично
надсилає повідомлення УСІМ авторизованим виконавцям із деталями:

  📋 Нова заявка #12
  🏠 Аудиторія: 305а
  👤 Заявник: Іванов І.І.
  📝 Опис: Не працює проектор

Під повідомленням дві кнопки:

  ✅ Прийняти
    • Заявка переходить у статус «В роботі»
    • Ваш Telegram ID записується як виконавець
    • Кнопки зникають у всіх виконавців
    • Усі отримують повідомлення, хто прийняв заявку
    • Заявка з'явиться у вашому списку «Мої заявки»

  ❌ Відхилити
    • Кнопки зникають лише у вас
    • Якщо ВСІ виконавці відхилили — заявка закривається
    • Якщо хтось інший прийме — вона перейде «В роботі»

═══════════════════════════════════════════════════════════════════
5. РОБОЧІ ПРОЦЕСИ (СЦЕНАРІЇ ВИКОРИСТАННЯ)
═══════════════════════════════════════════════════════════════════

─── Сценарій 1: Стандартна обробка заявки ───

  1. Заявник заповнює форму на сайті
  2. Бот надсилає сповіщення всім виконавцям
  3. Один із виконавців натискає «✅ Прийняти»
  4. Заявка → статус «В роботі», виконавець іде в аудиторію
  5. Після усунення проблеми статус змінюється на «Виконано»

─── Сценарій 2: Відхилення заявки ───

  1. Заявка надходить (наприклад, спам або помилка)
  2. Перший виконавець натискає «❌ Відхилити»
  3. Другий виконавець також натискає «❌ Відхилити»
  4. Якщо ВСІ відхилили → заявка автоматично закривається
  5. Усі отримують повідомлення: «Заявку відхилено усіма»

─── Сценарій 3: Перегляд своїх заявок ───

  1. Виконавець натискає /start або /tickets
  2. Натискає кнопку «📂 Мої заявки»
  3. Бачить список лише своїх заявок, що в роботі
  4. Натискає /ticket 12 для перегляду деталей конкретної

─── Сценарій 4: Моніторинг нових заявок ───

  1. Виконавець натискає /start або /tickets
  2. Натискає кнопку «🆕 Нові»
  3. Бачить усі заявки, що очікують обробки
  4. Може взяти будь-яку, дочекавшись сповіщення,
     або переглянути деталі через /ticket <номер>

═══════════════════════════════════════════════════════════════════
6. СТАТУСИ ЗАЯВОК
═══════════════════════════════════════════════════════════════════

  🟡 Нова        — Заявка створена, очікує на виконавця
  🔵 В роботі    — Виконавець прийняв та працює над проблемою
  🟢 Виконано    — Проблему усунено
  🔴 Закрито     — Відхилено усіма виконавцями

═══════════════════════════════════════════════════════════════════
7. ПРИМІТКИ
═══════════════════════════════════════════════════════════════════

• Бот працює лише для авторизованих Telegram ID.
  Додати нового виконавця: внесіть його ID у змінну
  ALLOWED_TELEGRAM_IDS у файлі docker-compose.yml
  та перезапустіть контейнер бота.

• Якщо один виконавець прийняв заявку, інші вже не можуть
  її прийняти або відхилити — кнопки зникають автоматично.

• Бот не зберігає стан між перезапусками. Якщо контейнер
  перезапущено, кнопки прийняття/відхилення на старих
  повідомленнях перестануть працювати (відповідь: «Заявка вже оброблена»).
